# Checklist de Entrega — Research de Pauta

Execute todos os itens antes de entregar ao `@head-de-conteudo`. Um único **NÃO** = pesquisa incompleta.

## Fontes de Dados

- [ ] Busca realizada no Google com a keyword exata?
- [ ] PAA (Pessoas Também Perguntaram) extraído?
- [ ] Buscas relacionadas (rodapé do Google) coletadas?
- [ ] Autocomplete analisado (sugestões enquanto se digita)?
- [ ] Top 3 resultados orgânicos analisados (H2s mapeados)?
- [ ] Gaps identificados nos top 2 resultados (o que não respondem)?

## Validação dos H2s

- [ ] Cada H2 tem fonte indicada (PAA / relacionadas / autocomplete / concorrentes)?
- [ ] Nenhum H2 foi inventado — todos rastreáveis a fonte real do Google?
- [ ] Nenhum H2 genérico proibido na lista final?
  - "O que é [tema]"
  - "Benefícios de [tema]"
  - "Como funciona [tema]"
  - "Vantagens e desvantagens"
  - "Dicas para [tema]"
  - "Conclusão"
- [ ] Cada H2 validado contra a Regra de Ouro: *"Essa pergunta aparece no Google quando alguém busca o tema?"*

## Output de Entrega

- [ ] Keyword principal confirmada (ou ajuste justificado)?
- [ ] Intenção de busca sugerida indicada?
- [ ] Variações semânticas identificadas (mínimo 2)?
- [ ] Formato de entrega padrão seguido (ver abaixo)?

## Formato Padrão de Entrega

```
PESQUISA — [keyword]

Keyword confirmada: [keyword]
Intenção sugerida: [Informacional | Comercial | Transacional]

H2s Validados:
1. [H2] — Fonte: PAA
2. [H2] — Fonte: PAA
3. [H2] — Fonte: buscas relacionadas
4. [H2] — Fonte: autocomplete
5. [H2] — Fonte: gap concorrente

Gaps identificados nos concorrentes:
- [Pergunta/tema que top 1 e top 2 não respondem]

Variações semânticas identificadas:
- [termo relacionado 1]
- [termo relacionado 2]
```
